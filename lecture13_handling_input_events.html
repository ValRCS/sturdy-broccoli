Browser Events

Events are actions or occurrences that happen in the system you are programming, which the system tells you about so you can respond to them in some way if desired. Your program can be said to be event-driven if it is designed to wait for events to occur rather than just plowing ahead with the computation.

In a web browser there are many different events that can fired/emmited by user actions and also external actions

For example:

  • Clicking a button
  • Clicking a link
  • Hovering over an element
  • Pressing a key
  • Scrolling
  • Resizing the window
  • Submitting a form
  • ...

Event Listeners

Event listeners are functions that are called when a specific event is fired. They are attached to a specific element and are called when the event is fired on that element.

For example, if you want to run a function when a button is clicked, you can add an event listener to the button that will run the function when the button is clicked.

Event listeners are added using the addEventListener() method. This method takes two arguments:

  • The name of the event to listen for
  • The function to run when the event is fired

Handling Input Events

HTML has various input types

For example:

  • Text input
  • Number input
  • Checkbox
  • Radio button
  • Dropdown
  • ...

Each of these input types have their own events that can be listened for.

For example:

  • Text input - input event
  • Number input - input event
  • Checkbox - change event
  • Radio button - change event
  • Dropdown - change event
  • ...
